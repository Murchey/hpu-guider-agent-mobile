根据我们在 `AiDialoguePage.vue` 中的代码逻辑，AI 可以通过发送特定格式的标签来触发前端的交互弹窗（单选卡和多选卡）。

---

## 1. 单选卡（场景/方案选择）
用于提供 1-3 个固定的方案供用户选择。

### 标签格式
使用 `[SCENE_DATA]` 标签包裹 JSON 数据。

### JSON 结构
```json
{
  "description": "这里是方案的简要说明，会显示在按钮上方的卡片中",
  "btn1": "按钮1文字",
  "btn2": "按钮2文字",
  "btn3": "按钮3文字"
}
```
- **description**：可选，字符串类型。用于在弹窗顶部展示一段说明文字。
- **字段名**：必须是 `btn1`、`btn2`、`btn3`（全部小写）。
- **可选性**：如果只需要 1 个或 2 个按钮，只需将多余的字段设为空字符串 `""` 或不写，前端会自动隐藏。

### AI 回复示例
```text
我已经为您筛选出了几个非常适合的景点。您可以点击下方按钮来生成详细的行程计划。

[SCENE_DATA]{"description": "以下是为您精心准备的三个行程方向，侧重点各有不同：", "btn1": "查看深度文化方案", "btn2": "查看休闲漫步方案", "btn3": "查看美食打卡方案"}[/SCENE_DATA]
```

---

## 2. 多选卡（景点/选项列表）
用于提供一个可多选的列表。

### 标签格式
使用 `[SELECT_DATA]` 标签包裹 JSON 数据。

### JSON 结构
```json
{
  "description": "这里是景点的简要介绍，会显示在列表上方的卡片中",
  "options": ["选项1", "选项2", "选项3", "选项4", "..."]
}
```
- **description**：可选，字符串类型。用于在弹窗顶部展示一段说明文字。
- **字段名**：必须是 `options`（数组类型）。
- **数量**：由于前端已增加**滚动条支持**，选项数量不再受限（建议不超过 15 个以保证体验）。

### AI 回复示例
```text
洛阳有很多值得一去的景点，我已经为您列出了几个热门选项，您可以勾选您感兴趣的景点，我将为您生成路线。

[SELECT_DATA]{"description": "这些景点都是洛阳最受游客欢迎的地方，涵盖了历史与人文：", "options": ["龙门石窟", "白马寺", "洛邑古城", "九州池"]}[/SELECT_DATA]
```

---

## 3. 用户反馈机制 (USER_CHOICE)
当用户在弹窗中做出选择后，前端会**自动隐藏发送**一条格式化的指令给 AI。AI 需要根据此指令进行下一步生成。

### 反馈格式示例
- **单选反馈**：`model:景点推荐\n[USER_CHOICE]{"value": "所选按钮文字"}[/USER_CHOICE]`
- **多选反馈**：`model:确定游览方案\n[USER_CHOICE]{"values": ["选项1", "选项2"]}[/USER_CHOICE]`

**AI 处理逻辑**：看到 `[USER_CHOICE]` 标签后，应根据 `model` 标识和 JSON 中的数据，立即结合之前的上下文（用户画像等）生成对应的详细方案。

---

## 4. 调用规范（必读）
- **JSON 标准**：必须使用**双引号**，严禁使用单引号。
- **多行文本**：如果 `description` 需要换行，必须使用 `\n` 进行转义。例如：`"description": "第一行\n第二行"`。
- **位置**：标签建议放在回复内容的**最后一行**，确保前端解析后不干扰正文阅读。
- **内容清理**：前端在解析成功后，会自动在对话流中隐藏 `[...]...[/...]` 部分，用户只会看到文字和弹出的对话框。
- **互斥性**：在同一次回复中，请只调用一种类型的卡片（单选或多选）。
